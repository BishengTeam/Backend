# 接口文档规范

> 适用项目：小程序平台后端（FastAPI）
> 更新日期：2026-05-16

---

## 一、输出格式

本项目使用 **OpenAPI 3.x**，由 FastAPI 自动生成 `/docs` 和 `/redoc`。

接口定稿阶段，前后端对齐使用 **Markdown 手写文档**，每个接口一张表，字段写到没有歧义为止。

---

## 二、接口文档模板

每个接口按以下模板描述：

```markdown
## {METHOD} {/api/path} — {一句话说明}

### 请求头
| Header | 值 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | Bearer <jwt_token> | 是 | 登录后获取的 Token |
| Content-Type | application/json | 是 | |

### 路径参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|

### 查询参数
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|

### 请求体
| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|

示例：
\```json
{
  "field": "value"
}
\```

### 响应体
| 字段 | 类型 | 说明 |
|------|------|------|

示例：
\```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
\```

### 错误码
| code | 说明 |
|------|------|
| 0 | 成功 |
| 40001 | 参数校验失败 |
| 40100 | Token 过期 |

### 示例
\```bash
curl -X POST "https://api.example.com/api/xxx" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"field": "value"}'
\```
```

## 三、统一约定

### 3.1 响应格式

所有接口统一返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

- `code=0` 表示成功，非 0 表示错误
- `message` 为人可读描述
- `data` 为业务数据，列表接口返回 `{ "items": [], "total": 100 }`

### 3.2 分页格式

列表接口统一使用：

```
Query: ?page=1&page_size=20
```

响应：

```json
{
  "code": 0,
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

### 3.3 命名风格

| 位置 | 风格 | 示例 |
|------|------|------|
| URL 路径 | kebab-case | `/api/quick-questions` |
| JSON 字段 | snake_case | `candidate_name` |
| 查询参数 | snake_case | `?page_size=20` |
| 枚举值 | snake_case | `"cert_type": "h3c"` |

### 3.4 时间格式

统一使用 ISO 8601 字符串：`"2026-05-16T14:30:00+08:00"`

纯日期使用：`"2026-05-16"`

### 3.5 金额字段

单位为**分**（整数），避免浮点精度问题。

```json
{ "price": 16800 }
```

## 四、错误码规范

### 4.1 错误码分段

| 范围 | 含义 |
|------|------|
| 0 | 成功 |
| 40001-40099 | 参数校验错误 |
| 40100-40199 | 认证/鉴权错误 |
| 40200-40299 | 业务逻辑错误（余额不足、状态不允许等） |
| 40300-40399 | 资源不存在 |
| 40400-40499 | 第三方服务错误（微信支付、OSS 等） |
| 50000-50099 | 服务器内部错误 |

### 4.2 错误响应格式

```json
{
  "code": 40001,
  "message": "参数校验失败",
  "detail": [
    { "field": "candidate_phone", "reason": "手机号格式不正确" }
  ]
}
```

`detail` 仅在参数校验失败时出现，用于前端定位具体字段。

---

## 五、鉴权约定

需要登录的接口请求头：

```
Authorization: Bearer <jwt_token>
```

- Token 有效期 2 小时
- 过期返回 `40101`
- Token 刷新接口：`POST /api/auth/refresh`

仅 `POST /api/auth/login` 及少数公开接口不需要 Token。

## 六、评审检查清单

接口文档定稿前，前后端逐项确认：

- [ ] URL 路径和方法无歧义
- [ ] 每个请求字段的类型、必填/可选明确
- [ ] 每个响应字段的含义可理解
- [ ] 错误码覆盖了本接口的异常场景
- [ ] 枚举值写全（不写「等」）
- [ ] 金额单位为分
- [ ] 时间格式为 ISO 8601
- [ ] 提供了一个可复制执行的 curl 示例
- [ ] 分页接口统一 `page` / `page_size`
- [ ] 前端无异议
