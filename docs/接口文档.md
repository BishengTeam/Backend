## POST /api/auth/login — 微信 code 登录，返回 JWT token

### 请求头

| Header | 值 | 必填 | 说明 |
|--------|------|------|------|
| Content-Type | application/json | 是 | |

### 路径参数

无

### 查询参数

无

### 请求体

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| code | string | 是 | — | 小程序调用 `wx.login()` 获取的临时凭证，5 分钟有效，一次性 |

示例：

```json
{
  "code": "081aBcDeFgHiJkLmN0oP1qRsT"
}
```

### 响应体

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0 |
| message | string | ok |
| data.access_token | string | JWT，接口鉴权用，有效期 2 小时 |
| data.refresh_token | string | 用于无感刷新 access_token，有效期 30 天 |
| data.expires_in | int | access_token 有效时长，单位秒 |
| data.user.id | int | 用户 ID |
| data.user.openid | string | 微信 openid |
| data.user.phone | string\|null | 手机号（未绑定则为 null） |
| data.user.created_at | string | 注册时间，ISO 8601 |

示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "rft_abc123def456...",
    "expires_in": 7200,
    "user": {
      "id": 1,
      "openid": "oABC123xyz...",
      "phone": null,
      "created_at": "2026-05-17T10:30:00+08:00"
    }
  }
}
```

### 错误码

| code | 说明 |
|------|------|
| 40001 | 参数校验失败（code 字段缺失或为空，详见 detail 字段） |
| 40100 | 该账号已注销，30 天内不可重新注册 |
| 40100 | 账号已注销，如需恢复请联系客服 |
| 50000 | 微信 code2session 失败或服务器内部错误 |

### 示例

```bash
curl -X POST "https://api.example.com/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"code": "081aBcDeFgHiJkLmN0oP1qRsT"}'
```

|--------|------|------|------|
| Content-Type | application/json | 是 | |

### 路径参数

无

### 查询参数

无

### 请求体

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| refresh_token | string | 是 | — | 登录或上次刷新时下发的 refresh_token |

示例：

```json
{
  "refresh_token": "rft_abc123def456..."
}
```

### 响应体

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0 |
| message | string | ok |
| data.access_token | string | 新签发的 JWT，有效期 2 小时 |
| data.refresh_token | string | 新签发的 refresh_token，旧 token 立即失效（轮换） |
| data.expires_in | int | access_token 有效时长，单位秒 |

示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "rft_new789ghi012...",
    "expires_in": 7200
  }
}
```

### 错误码

| code | 说明 |
|------|------|
| 40001 | 参数校验失败（refresh_token 字段缺失或为空） |
| 40100 | refresh_token 无效或已过期 |
| 40100 | 账号不存在或已注销 |

### 示例

```bash
curl -X POST "https://api.example.com/api/auth/refresh" \
  -H "Content-Type: application/json" \
  -d '{"refresh_token": "rft_abc123def456..."}'
```

无

### 查询参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| vendor | string | 否 | — | 按厂商筛选，可选值：`H3C` / `深信服` / `NISP` / `人社` |

### 请求体

无

### 响应体

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0 |
| message | string | ok |
| data[].id | int | 认证类型 ID |
| data[].name | string | 短名称 |
| data[].chinese_name | string | 中文全称 |
| data[].code | string | 考试代码，如 `H3C-NE` |
| data[].vendor | string | 厂商：`H3C` / `深信服` / `NISP` / `人社` |
| data[].requires_xuexin | bool | 报名是否需要学信网验证 |
| data[].pay_first | bool | 是否先支付后填写报名表 |

示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    {
      "id": 1,
      "name": "H3C-NE",
      "chinese_name": "H3C 网络工程师",
      "code": "H3C-NE",
      "vendor": "H3C",
      "requires_xuexin": false,
      "pay_first": false
    },
    {
      "id": 2,
      "name": "SF-CSE",
      "chinese_name": "深信服网络安全工程师",
      "code": "SF-CSE",
      "vendor": "深信服",
      "requires_xuexin": false,
      "pay_first": true
    }
  ]
}
```

### 错误码

| code | 说明 |
|------|------|
| 40001 | vendor 参数取值无效（不在枚举范围内） |
| 50000 | 服务器内部错误 |

### 示例

```bash
curl "https://api.example.com/api/cert/certifications"

curl "https://api.example.com/api/cert/certifications?vendor=H3C"
```

## POST /api/user/identity — 提交实名认证信息

### 请求头

| Header | 值 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | Bearer \<jwt_token\> | 是 | 登录后获取的 Token |
| Content-Type | application/json | 是 | |

### 路径参数

无

### 查询参数

无

### 请求体

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| real_name | string | 是 | — | 真实姓名 |
| id_card_number | string | 是 | — | 18 位身份证号，校验位须正确 |
| id_card_front_oss | string\|null | 否 | null | 身份证人像面 OSS key |
| id_card_back_oss | string\|null | 否 | null | 身份证国徽面 OSS key |
| student_card_oss | string\|null | 否 | null | 学生证 OSS key |

示例：

```json
{
  "real_name": "张三",
  "id_card_number": "320102199001011234",
  "id_card_front_oss": null,
  "id_card_back_oss": null,
  "student_card_oss": null
}
```

### 响应体

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0 |
| message | string | ok |
| data.real_name | string | 真实姓名 |
| data.id_card_number | string | 身份证号，脱敏返回（前 4 + 后 4，中间掩码） |
| data.id_card_front_oss | string\|null | 身份证人像面 OSS key |
| data.id_card_back_oss | string\|null | 身份证国徽面 OSS key |
| data.student_card_oss | string\|null | 学生证 OSS key |
| data.status | string | `verified`（提交即自动审核通过） |
| data.verified_at | string\|null | 审核通过时间，ISO 8601 |
| data.created_at | string | 创建时间，ISO 8601 |

示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "real_name": "张三",
    "id_card_number": "3201**********1234",
    "id_card_front_oss": null,
    "id_card_back_oss": null,
    "student_card_oss": null,
    "status": "verified",
    "verified_at": "2026-05-17T15:00:00+08:00",
    "created_at": "2026-05-17T15:00:00+08:00"
  }
}
```

### 错误码

| code | 说明 |
|------|------|
| 40001 | 参数校验失败（必填字段缺失） |
| 40001 | 身份证号格式不合法（长度 / 校验位） |
| 40100 | 未登录或 Token 已过期 |
| 40100 | 账号已注销 |
| 40200 | 实名信息最多修改 3 次，已达上限 |
| 40300 | 用户不存在 |

### 示例

```bash
curl -X POST "https://api.example.com/api/user/identity" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -H "Content-Type: application/json" \
  -d '{"real_name":"张三","id_card_number":"320102199001011234"}'
```

## GET /api/chat/stream — SSE 流式消息

### 请求头

| Header | 值 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | Bearer \<jwt_token\> | 是 | 登录后获取的 Token |

### 路径参数

无

### 查询参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| message | string | 是 | — | 用户消息文本，1-2000 字符 |
| session_id | string | 否 | — | 会话 ID，首次为空则由服务端创建 |

### 请求体

无

### 响应体

SSE 流（`text/event-stream`），每条事件格式：

```
data: {"chunk": "<文本片段>", "session_id": "<session_id>"}
```

流结束：

```
data: {"done": true, "session_id": "<session_id>", "reply": "<完整回复文本>"}
```

异常时：

```
data: {"error": "服务异常，请重试", "session_id": "<session_id>"}
```

示例流：

```
data: {"chunk": "您好", "session_id": "a1b2c3d4-..."}

data: {"chunk": "，", "session_id": "a1b2c3d4-..."}

data: {"chunk": "人工", "session_id": "a1b2c3d4-..."}

data: {"chunk": "客服", "session_id": "a1b2c3d4-..."}

data: {"chunk": "正在", "session_id": "a1b2c3d4-..."}

data: {"chunk": "赶来", "session_id": "a1b2c3d4-..."}

data: {"chunk": "，", "session_id": "a1b2c3d4-..."}

data: {"chunk": "请稍候", "session_id": "a1b2c3d4-..."}

data: {"chunk": "...", "session_id": "a1b2c3d4-..."}

data: {"done": true, "session_id": "a1b2c3d4-...", "reply": "您好，人工客服正在赶来，请稍候..."}
```

### 错误码

| code | 说明 |
|------|------|
| 40001 | 参数校验失败（message 字段缺失或为空） |
| 40100 | 未登录或 Token 已过期 |
| 50000 | 服务器内部错误 |

> 流内异常通过 `{"error": "..."}` 事件推送，不走 JSON 错误响应。

### 示例

```bash
curl "https://api.example.com/api/chat/stream?message=%E4%BD%A0%E5%A5%BD" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."
```
