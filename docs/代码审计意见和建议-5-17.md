## 二、逐项审计详情

### A. 成功 code 使用 HTTP 状态码而非 0（6 处）

规范 3.1 节：`code=0` 表示成功。代码中所有地方都写成 `200`：

| 位置 | 当前值 | 应改为 |
|------|--------|--------|
| `app/schemas/common.py:9` APIResponse 默认值 | `code: int = 200` | `code: int = 0` |
| `app/schemas/common.py:26` success() | `code: int = 200` | `code: int = 0` |
| `app/schemas/common.py:30` created() | `code=201` | `code=0` |
| `app/main.py:33` health 接口 | `"code": 200` | `"code": 0` |

### B. 错误码混用 HTTP 状态码（5 个异常类）

规范 4.1 节定义了 5 位分段错误码，但代码复用了 HTTP 状态码：

| 异常类 | 当前 code | 应改为 | 对应 HTTP status |
|--------|-----------|--------|-------------------|
| `NotFoundException` | 404 | 40300 | 404 |
| `BusinessException` | 422 | 40200 | 422 |
| `UnauthorizedException` | 401 | 40100 | 401 |
| `ForbiddenException` | 403 | 40101 | 403 |
| `ConflictException` | 409 | 40201 | 409 |
| 全局异常兜底 | 500 | 50000 | 500 |

### C. 响应体未用 data 包装（6 个接口）

规范 3.1 节：业务数据必须放在 `data` 字段内。以下接口裸返了 schema：

| 接口 | 当前 response_model | 客户端看到 | 应改为 |
|------|---------------------|-----------|--------|
| `POST /api/auth/login` | `LoginResponse` | `{access_token, user, ...}` | 包在 data 内 |
| `POST /api/auth/refresh` | `RefreshResponse` | `{access_token, ...}` | 包在 data 内 |
| `GET /api/user/profile` | `UserProfile` | `{id, openid, ...}` | 包在 data 内 |
| `PUT /api/user/profile` | `UserProfile` | `{id, openid, ...}` | 包在 data 内 |
| `GET /api/cert/certifications` | `list[CertificationResponse]` | `[{...}, ...]` | 包在 data 内 |
| `GET /api/cert/xuexin` | `XuexinGuideResponse` | `{title, steps, ...}` | 包在 data 内 |
| `POST /api/chat` | `ChatResponse` | `{session_id, reply}` | 包在 data 内 |
| `GET /api/quick-questions` | `list[QuickQuestionResponse]` | `[{...}, ...]` | 包在 data 内 |

已符合的接口（用了 `success()` 包装）：`DELETE /api/user/account`、`POST /api/user/phone/decrypt`、`GET /api/system/poster`。

### D. JWT 过期时间配置异常

- 文档写 access_token 有效期 **2 小时**
- 代码 `app/core/config.py:21` → `JWT_EXPIRE_MINUTES: int = 43200`（30 天）
- 文档中 `expires_in` 示例值为 7200，但实际返回 2,592,000

应改为 `JWT_EXPIRE_MINUTES: int = 120`。

### E. FastAPI 校验错误未适配规范格式

FastAPI 默认返回：
```json
{"detail": [{"loc": ["body", "code"], "msg": "field required", "type": "value_error.missing"}]}
```

规范 4.2 节要求：
```json
{"code": 40001, "message": "参数校验失败", "detail": [{"field": "code", "reason": "微信登录 code 不能为空"}]}
```

需新增 `RequestValidationError` 的异常处理器。

### F. SecurityMiddleware 已定义但未注册

`app/middleware/security.py` 实现了 SQL 注入和 XSS 检测，但在 `app/middleware/__init__.py` 的 `setup_middleware` 中未调用。需补注册。

### 变更 1：`app/core/exceptions.py`

```python
class AppException(Exception):
    def __init__(self, code: int, message: str, http_status_code: int = 400):
        self.code = code          # 业务错误码（5 位分段码）
        self.message = message
        self.http_status_code = http_status_code  # HTTP 状态码


class NotFoundException(AppException):
    def __init__(self, resource: str):
        super().__init__(code=40300, message=f"{resource} 不存在", http_status_code=404)


class BusinessException(AppException):
    def __init__(self, message: str):
        super().__init__(code=40200, message=message, http_status_code=422)


class UnauthorizedException(AppException):
    def __init__(self, message: str = "请先登录"):
        super().__init__(code=40100, message=message, http_status_code=401)


class ForbiddenException(AppException):
    def __init__(self, message: str = "无权限"):
        super().__init__(code=40101, message=message, http_status_code=403)


class ConflictException(AppException):
    def __init__(self, message: str):
        super().__init__(code=40201, message=message, http_status_code=409)


class ValidationException(AppException):
    def __init__(self, message: str = "参数校验失败", detail: list[dict] | None = None):
        super().__init__(code=40001, message=message, http_status_code=422)
        self.detail = detail or []
```

### 变更 2：`app/middleware/error_handler.py`

```python
from fastapi import Request
from fastapi.exceptions import RequestValidationError
from fastapi.responses import JSONResponse

from app.core.exceptions import AppException


async def app_exception_handler(request: Request, exc: AppException) -> JSONResponse:
    content = {"code": exc.code, "message": exc.message, "data": None}
    if hasattr(exc, "detail") and exc.detail:
        content["detail"] = exc.detail
    return JSONResponse(status_code=exc.http_status_code, content=content)


async def validation_exception_handler(request: Request, exc: RequestValidationError) -> JSONResponse:
    detail = []
    for error in exc.errors():
        field = ".".join(str(loc) for loc in error["loc"] if loc not in ("body", "query", "path"))
        detail.append({"field": field, "reason": error["msg"]})
    return JSONResponse(
        status_code=422,
        content={"code": 40001, "message": "参数校验失败", "detail": detail},
    )


async def global_exception_handler(request: Request, exc: Exception) -> JSONResponse:
    return JSONResponse(
        status_code=500,
        content={"code": 50000, "message": "服务器内部错误", "data": None},
    )
```

同时修改 `app/middleware/__init__.py`，注册 validation_exception_handler 和新异常处理器。

### 变更 3：`app/schemas/common.py`

```python
class APIResponse(BaseModel, Generic[T]):
    code: int = 0
    message: str = "ok"
    data: T | None = None

def success(data: Any = None, message: str = "ok", code: int = 0) -> APIResponse:
    return APIResponse(code=code, message=message, data=data)

def created(data: Any = None, message: str = "创建成功") -> APIResponse:
    return APIResponse(code=0, message=message, data=data)
```

### 变更 11：`app/middleware/__init__.py`

def setup_middleware(app: FastAPI) -> None:
    setup_cors(app)
    app.add_middleware(SecurityMiddleware)
    app.add_middleware(RequestIDMiddleware)

## 四、影响评估

| 维度 | 影响 | 说明 |
|------|------|------|
| 前端 | **破坏性变更** | `code` 判断、响应解析路径均需同步调整 |
| 移动端/小程序 | **破坏性变更** | 同上，需协调发版 |
| JWT 会话 | **破坏性变更** | 已签发的 30 天 token 缩为 2 小时，存量 session 会提前失效 |
| 监控/日志 | 低 | health 返回体字段值变化，如有告警规则需更新 |
| SecurityMiddleware | 低风险 | 首次启用，可能误拦截含合法 SQL/XSS 特征字符串的请求（如密码含 `'OR'`) |

---

## 五、建议上线顺序

1. 前端/小程序先适配新的响应格式（兼容期读取 `response.data.xxx` + fallback `response.xxx`）
2. 后端上线规范修复版本
3. 前端/小程序切到新路径，去掉兼容逻辑
4. 监控验证 24h 无异常后关闭
